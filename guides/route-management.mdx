## Descripción General

Las rutas son la base de la operación de ventas en Fabrica Marie ERP. Una ruta define el recorrido que un vendedor realizará, los clientes que visitará y el orden de las visitas. Esta guía te ayudará a crear y gestionar rutas eficientemente.

<Info>
  **Roles con acceso:** ADMIN, GERENTE, SUPERVISOR, VENDEDOR, FIDELIZACION, MANTENIMIENTO
</Info>

## Conceptos Fundamentales

### ¿Qué es una Ruta?

Una ruta es un conjunto ordenado de clientes que un vendedor visita regularmente. Cada ruta tiene:

- **Nombre identificativo**: Ej: "Ruta Norte A", "Ruta Centro 1"
- **Descripción**: Detalle de la zona o características
- **Clientes asignados**: Lista de clientes con orden de visita
- **Días de operación**: Días de la semana que se recorre
- **Estado**: Activa o inactiva

<Steps>
  <Step title="Definir zona y características">

### Actualizar Ruta

```bash
PUT /api/rutas/{id}
Authorization: Bearer {token}
Content-Type: application/json

{
  "nombre": "Ruta Norte A - Ampliada",
  "descripcion": "Zona norte ampliada con nuevos clientes",
  "dias_operacion": "Lunes, Miércoles, Viernes, Sábado"
```

### Desactivar Ruta

En lugar de eliminar, desactiva rutas que ya no se usan:

```bash
DELETE /api/rutas/{id}
Authorization: Bearer {token}
```

Esto cambia `activo` a `false`, manteniendo el historial.

## Asignación de Clientes

### Asignar Clientes a Ruta

La asignación de clientes incluye un **orden de visita** que define la secuencia del recorrido:

<Steps>
  <Step title="Planificar orden de visitas">
    Determina el orden óptimo considerando:
    - Proximidad geográfica
    - Horarios de atención
    - Volumen de compra
    - Prioridad del cliente
  </Step>

  <Step title="Asignar clientes con orden">
    ```bash
    POST /api/rutas/{ruta_id}/asignar-clientes
    Authorization: Bearer {token}
    Content-Type: application/json

    {
      "clientes": [
        {
          "cliente_id": 15,
          "orden": 1
        },
        {
          "cliente_id": 18,
          "orden": 2
        },
        {
          "cliente_id": 22,
          "orden": 3
        },
        {
          "cliente_id": 30,
          "orden": 4
        },
        {
          "cliente_id": 35,
          "orden": 5
        }
      ]
    }
    ```
  </Step>

  <Step title="Recibir confirmación">
    ```json
    {
      "message": "Clientes asignados a la ruta"
    }
    ```
  </Step>
</Steps>

<Note>
  El sistema utiliza `sync()`, lo que significa que reemplaza la asignación anterior. Si quieres mantener clientes existentes, debes incluirlos en la nueva lista.
</Note>

### Reordenar Clientes

Para cambiar el orden de visitas, simplemente reasigna con el nuevo orden:

```bash
POST /api/rutas/8/asignar-clientes
Authorization: Bearer {token}

{
  "clientes": [
    { "cliente_id": 18, "orden": 1 },  // Ahora es el primero
    { "cliente_id": 15, "orden": 2 },  // Ahora es el segundo
    { "cliente_id": 22, "orden": 3 }
  ]
}
```

### Agregar Cliente a Ruta Existente

Para agregar un cliente sin afectar los demás:

1. Consulta los clientes actuales de la ruta
2. Incluye todos los clientes existentes más el nuevo
3. Reasigna con el nuevo orden

## Operación de Rutas

### Flujo Diario de una Ruta

<Steps>
  <Step title="Planificación (Día Anterior)">
    - Gerencia revisa rutas programadas para mañana
    - Asigna vendedor y vehículo a cada ruta
    - Planifica carga de productos
  </Step>

  <Step title="Preparación (7:00 AM)">
    - Almacén prepara salida con productos de la ruta
    - Asigna vendedor y vehículo
    - Genera documentación
    
    ```bash
    POST /api/inventario/salidas
    {
      "fecha": "2026-03-11",
      "vendedor_id": 5,
      "vehiculo_id": 2,
      "ruta_id": 8,
      "zona": "NORTE",
      "items": [...]
    }
    ```
  </Step>

  <Step title="Inicio de Recorrido (8:00 AM)">
    - Vendedor recibe productos y vehículo
    - Actualiza estado de salida a EN_RUTA
    - Inicia GPS tracking
    
    ```bash
    PUT /api/inventario/salidas/estado/45
    { "estado": "EN_RUTA" }
    ```
  </Step>

  <Step title="Visitas a Clientes (8:00 AM - 5:00 PM)">
    Para cada cliente en orden:
    1. Llegar a ubicación del cliente
    2. Mostrar productos disponibles
    3. Tomar pedido
    4. Crear y confirmar venta
    5. Cobrar (contado) o abonar (crédito)
    6. Continuar al siguiente cliente
  </Step>

  <Step title="Fin de Recorrido (5:30 PM)">
    - Completar salida
    - Regresar a fábrica
    - Entregar productos devueltos
    - Entregar efectivo a caja
    
    ```bash
    PUT /api/inventario/salidas/estado/45
    { "estado": "COMPLETADO" }
    ```
  </Step>
</Steps>

```bash
Authorization: Bearer {token}
```

### Analizar Rendimiento de Ruta

```bash
GET /api/rutas/{id}/rendimiento?mes=2026-03
Authorization: Bearer {token}
```

**Métricas útiles:**
- Total vendido en el mes
- Promedio de venta por visita
- Número de clientes visitados
- Tasa de conversión (visitas vs ventas)
- Productos más vendidos en la ruta
- Clientes top de la ruta

### Mejores Prácticas

### Factores a Considerar

1. **Distancia total del recorrido**: Minimiza kilómetros
2. **Tiempo de viaje**: Considera tráfico y condiciones
3. **Capacidad del vehículo**: No sobrecargues
4. **Horarios de clientes**: Respeta horarios de atención
5. **Potencial de venta**: Prioriza clientes importantes
6. **Créditos pendientes**: Programa cobros estratégicamente

### Ejemplo de Ruta Optimizada

```

1. Bodega El Sol (8:30 AM) - Cliente grande, alta prioridad
2. Bodega Miñita (9:00 AM) - Mismo sector

4. Bodega La Esquina (10:00 AM) - Vecina
...
25. Bodega Nocturna (5:00 PM) - Última parada antes de retorno

Tiempo total: 8.5 horas
Distancia: 45 km
Ventas esperadas: S/ 6,500
```

## Código de Referencia

**RutaController.php** (`app/Http/Controllers/Api/RutaController.php:51-72`)

```php
// Asignar clientes con orden
public function asignarClientes(Request $request, $rutaId)
{
    $ruta = Ruta::findOrFail($rutaId);

    $syncData = [];
    foreach ($request->clientes as $item) {
        $syncData[$item['cliente_id']] = ['orden' => $item['orden']];
    }

    $ruta->clientes()->sync($syncData);

    return response()->json(['message' => 'Clientes asignados a la ruta']);
}
```
