## Listar Usuarios

<api-endpoint method="GET" endpoint="/api/admin/usuarios" />

Obtiene la lista de todos los usuarios activos del sistema con sus roles asignados.

### Headers

<ParamField header="Authorization" type="string" required>
  Bearer token de autenticación obtenido del login
</ParamField>

### Permisos Requeridos

- Rol: `ADMIN`

### Response

<ResponseField name="data" type="array">
  Lista de usuarios del sistema
  
  <ResponseField name="id" type="integer">
    Identificador único del usuario
  </ResponseField>
  
  <ResponseField name="username" type="string">
    Nombre de usuario para acceso al sistema
  </ResponseField>
  
  <ResponseField name="email" type="string">
    Correo electrónico del usuario
  </ResponseField>
  
  <ResponseField name="nombre" type="string">
    Nombre completo del usuario
  </ResponseField>
  
  <ResponseField name="activo" type="boolean">
    Estado del usuario (1 = activo, 0 = inactivo)
  </ResponseField>
  
  <ResponseField name="created_at" type="string">
    Fecha de creación del usuario
  </ResponseField>
  
  <ResponseField name="roles" type="array">
    Roles asignados al usuario
    
    <ResponseField name="id" type="integer">
      Identificador del rol
    </ResponseField>
    
    <ResponseField name="nombre" type="string">
      Nombre del rol (ej: ADMIN, VENDEDOR, ALMACENERO)
    </ResponseField>
  </ResponseField>
</ResponseField>

<CodeGroup>

```bash cURL
curl -X GET \
  'https://api.fabricamarie.com/api/admin/usuarios' \
  -H 'Authorization: Bearer {token}' \
  -H 'Accept: application/json'
```

```php PHP

$response = $client->get('https://api.fabricamarie.com/api/admin/usuarios', [
  'headers' => [
    'Authorization' => 'Bearer ' . $token,
    'Accept' => 'application/json'
  ]
]);

```

```javascript JavaScript

    'Accept': 'application/json'
  }
});

const usuarios = await response.json();
```

## Crear Usuario

<api-endpoint method="POST" endpoint="/api/admin/usuarios" />

Crea un nuevo usuario en el sistema con su información personal, roles y datos salariales. Si se asigna el rol de VENDEDOR, automáticamente se crea un registro de vendedor asociado.

### Headers

<ParamField header="Authorization" type="string" required>
  Bearer token de autenticación obtenido del login
</ParamField>

<ParamField header="Content-Type" type="string" required>
  application/json
</ParamField>

### Permisos Requeridos

- Rol: `ADMIN`

### Body Parameters

<ParamField body="username" type="string" required>
  Nombre de usuario único para el acceso al sistema. Debe ser único en la base de datos.
</ParamField>

<ParamField body="email" type="string" required>
  Correo electrónico del usuario. Debe ser un email válido y único en el sistema.
</ParamField>

<ParamField body="nombre" type="string" required>
  Nombre completo del usuario
</ParamField>

<ParamField body="password" type="string" required>
  Contraseña del usuario. Debe tener al menos 6 caracteres.
</ParamField>

<ParamField body="roles" type="array" required>
  Array de IDs de roles a asignar al usuario. Debe contener al menos un rol.
  
  Roles disponibles:
  - `1`: ADMIN
  - `2`: GERENTE
  - `3`: VENDEDOR
  - `4`: ALMACENERO
  - `5`: CAJERO
  - `6`: SUPERVISOR
  - `7`: RRHH
  - `8`: FIDELIZACION
  - `9`: MANTENIMIENTO
</ParamField>

<ParamField body="sueldo_base" type="number" required>
  Sueldo base mensual del empleado. Debe ser mayor o igual a 0.
</ParamField>

<ParamField body="horas_extra" type="number" required>
  Tarifa por hora extra del empleado. Debe ser mayor o igual a 0.
</ParamField>

<ParamField body="afp" type="number" required>
  Porcentaje o monto de AFP (Administradora de Fondos de Pensiones) del empleado. Debe ser mayor o igual a 0.
</ParamField>

### Response

<ResponseField name="message" type="string">
  Mensaje de confirmación de la creación del usuario
</ResponseField>

<CodeGroup>
```json Request Body
{
  "username": "mrodriguez",
  "email": "mrodriguez@fabricamarie.com",
  "nombre": "María Rodríguez",
  "password": "password123",
  "roles": [3],
  "sueldo_base": 2500.00,
  "horas_extra": 15.50,
  "afp": 325.00
}
```

```json Response 201
{
  "message": "Usuario creado correctamente"
}
```

```json Error 422 - Validación
{
  "message": "The username has already been taken.",
  "errors": {
    "username": [
      "The username has already been taken."
    ]
  }
}
```

```bash cURL
  -H 'Authorization: Bearer {token}' \
  -H 'Content-Type: application/json' \

  }'
```

</CodeGroup>

### Notas Importantes

<Note>
  **Creación Automática de Vendedor**: Si se asigna el rol `VENDEDOR` (ID: 3), el sistema automáticamente creará un registro en la tabla de vendedores asociado al usuario creado.
</Note>

<Warning>
  **Información Salarial**: Los datos salariales (sueldo_base, horas_extra, afp) son obligatorios y se almacenan en una tabla separada de información salarial vinculada al usuario.
</Warning>

<Info>
  **Contraseña**: La contraseña se almacena hasheada usando el algoritmo bcrypt de Laravel. Nunca se almacena en texto plano.
</Info>

## Listar Vendedores

<api-endpoint method="GET" endpoint="/api/vendedores" />

Obtiene la lista de todos los vendedores activos con su información de usuario asociada.

### Headers

<ParamField header="Authorization" type="string" required>
  Bearer token de autenticación obtenido del login
</ParamField>

### Permisos Requeridos

- Roles permitidos: Cualquier usuario autenticado

### Response

<ResponseField name="data" type="array">
  Lista de vendedores activos
  
  <ResponseField name="id" type="integer">
    Identificador único del vendedor
  </ResponseField>
  
  <ResponseField name="usuario_id" type="integer">
    ID del usuario asociado al vendedor
  </ResponseField>
  
  <ResponseField name="activo" type="boolean">
    Estado del vendedor (1 = activo, 0 = inactivo)
  </ResponseField>
  
  <ResponseField name="usuario" type="object">
    Información del usuario asociado
    
    <ResponseField name="id" type="integer">
      ID del usuario
    </ResponseField>
    
    <ResponseField name="username" type="string">
      Nombre de usuario
    </ResponseField>
    
    <ResponseField name="nombre" type="string">
      Nombre completo del vendedor
    </ResponseField>
    
    <ResponseField name="email" type="string">
      Correo electrónico
    </ResponseField>
  </ResponseField>
</ResponseField>
