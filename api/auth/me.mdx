## Descripción

Retorna la información completa del usuario actualmente autenticado, incluyendo sus roles y permisos. Este endpoint es útil para:

- Verificar si el token sigue siendo válido
- Obtener información actualizada del usuario
- Validar permisos antes de mostrar funcionalidades
- Refrescar el estado de la sesión

<Warning>
**Requiere autenticación:** Este endpoint requiere un token válido de Sanctum en el header `Authorization`.
</Warning>

## Autenticación

```bash
Authorization: Bearer {token}
```

Donde `{token}` es el token obtenido del endpoint `/api/login`.

## Parámetros

Este endpoint no requiere parámetros.

## Errores

### Códigos de error comunes

<ResponseExample>
```json 401 - No autenticado
{
  "message": "Unauthenticated."
}
```
</ResponseExample>

<Note>
El código de error 401 se retorna cuando:
- No se proporciona el header `Authorization`
- El token es inválido o ha expirado
- El token fue eliminado (ej: después de un logout)
</Note>

const token = localStorage.getItem('auth_token');

const response = await fetch('https://api.fabricamarie.com/api/me', {
  method: 'GET',

    'Authorization': `Bearer ${token}`,
    'Accept': 'application/json'

  console.log('Usuario:', data.user);
  console.log('Roles:', data.roles);
  console.log('Permisos:', data.permisos);
  
  // Verificar si tiene un permiso específico
  const puedeCrearRol = data.permisos.some(
    permiso => permiso.nombre === 'crear_rol'
  );
  
  // Verificar si tiene un rol específico
  const esAdmin = data.roles.some(
    rol => rol.nombre === 'ADMIN'
  );
  
  console.log('Es admin:', esAdmin);
  console.log('Puede crear rol:', puedeCrearRol);
} else {
  console.error('Token inválido o expirado');
  // Redirigir al login
  window.location.href = '/login';
}

## Casos de uso

### 1. Verificar validez del token

```javascript
async function verificarSesion() {
  try {
    const response = await fetch('/api/me', {
      headers: {
        'Authorization': `Bearer ${token}`,
        'Accept': 'application/json'
      }
    });
    
    if (!response.ok) {
      // Token inválido, redirigir al login
      window.location.href = '/login';
    }
  } catch (error) {
    console.error('Error verificando sesión:', error);
  }
}
```

### 2. Validar permisos para funcionalidades

```javascript
const data = await fetch('/api/me').then(r => r.json());

// Mostrar botón solo si tiene el permiso
const puedeCrearVenta = data.permisos.some(
  p => p.nombre === 'crear_venta'
);

if (puedeCrearVenta) {
  mostrarBotonNuevaVenta();
}
```

### 3. Refrescar información del usuario

```javascript
// Después de que un admin actualice los permisos del usuario
async function refrescarPermisos() {
  const response = await fetch('/api/me', {
    headers: { 'Authorization': `Bearer ${token}` }
  });
  
  const data = await response.json();
  actualizarInterfazConNuevosPermisos(data.permisos);
}
```

## Notas adicionales

<Info>
**Usos recomendados:**

- Llamar este endpoint al cargar la aplicación para validar la sesión
- Utilizar la información de roles/permisos para mostrar/ocultar funcionalidades en el frontend
- Implementar un sistema de "refresh" de permisos sin requerir nuevo login
- Validar que el usuario sigue activo (`activo: 1`)
</Info>

<Tip>
**Optimización:**

Puedes cachear la respuesta de este endpoint durante un tiempo razonable (ej: 5-10 minutos) para evitar múltiples llamadas innecesarias. Sin embargo, asegúrate de refrescar la información cuando:

- El usuario realiza una acción que podría cambiar sus permisos
- Después de un período de inactividad
- Al navegar entre secciones críticas que requieren verificación de permisos
</Tip>

<Warning>
**Seguridad:**

Nunca confíes únicamente en la validación del frontend. Aunque uses este endpoint para mostrar/ocultar elementos de la UI, el backend siempre debe validar los permisos en cada petición a endpoints protegidos.
</Warning>
