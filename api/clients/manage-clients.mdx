## Listar Clientes

<Note>
  Obtiene la lista de todos los clientes activos con su información de ruta asociada.
</Note>

### Endpoint

```
GET /api/clientes
```

### Autenticación

<ParamField header="Authorization" type="string" required>
  Token de autenticación Bearer
</ParamField>

### Respuesta

Retorna un array de clientes activos.

<ResponseField name="[]" type="array">
  Array de objetos de cliente
  
  <Expandable title="Propiedades de cada cliente">
    <ResponseField name="id" type="integer">
      ID único del cliente
    </ResponseField>
    
    <ResponseField name="codigo_cliente" type="string">
      Código único del cliente
    </ResponseField>
    
    <ResponseField name="razon_social" type="string">
      Nombre o razón social del cliente
    </ResponseField>
    
    <ResponseField name="tipo_cliente" type="string">
      Tipo de cliente: TIENDA, DISTRIBUIDOR, MAYORISTA, CONSUMIDOR
    </ResponseField>
    
    <ResponseField name="direccion" type="string">
      Dirección del cliente
    </ResponseField>
    
    <ResponseField name="telefono" type="string">
      Teléfono de contacto
    </ResponseField>
    
    <ResponseField name="ruta_id" type="integer">
      ID de la ruta asignada
    </ResponseField>
    
    <ResponseField name="condicion_pago" type="string">
      Condición de pago: CONTADO o CREDITO
    </ResponseField>
    
    <ResponseField name="limite_credito" type="number">
      Límite de crédito autorizado
    </ResponseField>
    
    <ResponseField name="dias_credito" type="integer">
      Días de plazo para crédito
    </ResponseField>
    
    <ResponseField name="deuda_actual" type="number">
      Monto de deuda actual
    </ResponseField>
    
    <ResponseField name="activo" type="boolean">
      Estado del cliente (true = activo)
    </ResponseField>
    
    <ResponseField name="status" type="string">
      Estado del cliente
    </ResponseField>
    
    <ResponseField name="ruta" type="object">
      Información de la ruta asignada (si existe)
    </ResponseField>
  </Expandable>
</ResponseField>

## Crear Cliente

<Note>
  Crea un nuevo cliente en el sistema con opción de asignarlo a una ruta.
</Note>

### Endpoint

```
POST /api/clientes
```

### Autenticación

<ParamField header="Authorization" type="string" required>
  Token de autenticación Bearer
</ParamField>

### Parámetros del Body

<ParamField body="codigo_cliente" type="string" required>
  Código único del cliente (debe ser único en el sistema)
</ParamField>

<ParamField body="razon_social" type="string" required>
  Nombre o razón social del cliente
</ParamField>

<ParamField body="tipo_cliente" type="string">
  Tipo de cliente. Valores permitidos: `TIENDA`, `DISTRIBUIDOR`, `MAYORISTA`, `CONSUMIDOR`
</ParamField>

<ParamField body="direccion" type="string">
  Dirección del cliente
</ParamField>

<ParamField body="telefono" type="string">
  Teléfono de contacto
</ParamField>

<ParamField body="ruta_id" type="integer">
  ID de la ruta a la que se asignará el cliente (debe existir en la tabla rutas)
</ParamField>

<ParamField body="condicion_pago" type="string" required>
  Condición de pago. Valores permitidos: `CONTADO`, `CREDITO`
</ParamField>

<ParamField body="limite_credito" type="number">
  Límite de crédito autorizado (requerido si condicion_pago es CREDITO)
</ParamField>

<ParamField body="dias_credito" type="integer">
  Días de plazo para pago a crédito
</ParamField>

<ParamField body="deuda_actual" type="number">
  Deuda actual del cliente (default: 0)
</ParamField>

<ParamField body="activo" type="boolean">
  Estado del cliente (default: true)
</ParamField>

<ParamField body="status" type="string" required>
  Estado textual del cliente
</ParamField>

### Comportamiento

- Si se proporciona `ruta_id`, el cliente se asigna automáticamente a esa ruta con orden 0
- El código de cliente debe ser único en el sistema
- La operación se ejecuta en una transacción de base de datos

## Ver Detalles de Cliente

<Note>
  Obtiene información completa de un cliente específico incluyendo sus rutas.
</Note>

### Endpoint

```
GET /api/clientes/{id}
```

### Autenticación

<ParamField header="Authorization" type="string" required>
  Token de autenticación Bearer
</ParamField>

### Parámetros de Ruta

<ParamField path="id" type="integer" required>
  ID del cliente a consultar
</ParamField>

### Respuesta

<ResponseField name="id" type="integer">
  ID del cliente
</ResponseField>

<ResponseField name="ruta" type="object">
  Información de la ruta principal asignada
</ResponseField>

<ResponseField name="rutas" type="array">
  Lista de todas las rutas en las que participa el cliente con información del pivot (orden)
</ResponseField>

## Desactivar Cliente

<Warning>
  Desactiva un cliente en lugar de eliminarlo físicamente del sistema. El cliente permanece en la base de datos pero con estado inactivo.
</Warning>

### Endpoint

```
DELETE /api/clientes/{id}
```

### Autenticación

<ParamField header="Authorization" type="string" required>
  Token de autenticación Bearer
</ParamField>

### Parámetros de Ruta

<ParamField path="id" type="integer" required>
  ID del cliente a desactivar
</ParamField>

### Comportamiento

- No elimina físicamente el registro
- Establece el campo `activo` en `false`
- El cliente no aparecerá en el listado de clientes activos
- Los registros históricos relacionados se mantienen intactos

<CodeGroup>

```bash Solicitud
curl -X DELETE "https://api.fabricamarie.com/api/clientes/45" \
  -H "Authorization: Bearer {token}"
```

```json Respuesta 200 OK
{
  "message": "Cliente desactivado"
}
```

```json Error 404 - No encontrado
{
  "message": "No query results for model [App\\Models\\Cliente] 45"
}
```

</CodeGroup>
