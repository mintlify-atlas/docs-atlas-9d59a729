## Listar Rutas

<Note>
  Obtiene la lista de todas las rutas activas con conteo de clientes asignados.
</Note>

### Endpoint

```
GET /api/rutas
```

### Autenticación

<ParamField header="Authorization" type="string" required>
  Token de autenticación Bearer
</ParamField>

### Respuesta

Retorna un array de rutas activas.

<ResponseField name="[]" type="array">
  Array de objetos de ruta
  
  <Expandable title="Propiedades de cada ruta">
    <ResponseField name="id" type="integer">
      ID único de la ruta
    </ResponseField>
    
    <ResponseField name="nombre" type="string">
      Nombre de la ruta
    </ResponseField>
    
    <ResponseField name="descripcion" type="string">
      Descripción de la ruta
    </ResponseField>
    
    <ResponseField name="activo" type="boolean">
      Estado de la ruta (true = activa)
    </ResponseField>
    
    <ResponseField name="clientes_count" type="integer">
      Número de clientes asignados a esta ruta
    </ResponseField>
    
    <ResponseField name="clientes" type="array">
      Lista de clientes asignados con información completa
    </ResponseField>
  </Expandable>
</ResponseField>

## Desactivar Ruta

<Warning>
  Desactiva una ruta en lugar de eliminarla físicamente del sistema.
</Warning>

### Endpoint

```
DELETE /api/rutas/{id}
```

### Autenticación

<ParamField header="Authorization" type="string" required>
  Token de autenticación Bearer
</ParamField>

### Parámetros de Ruta

<ParamField path="id" type="integer" required>
  ID de la ruta a desactivar
</ParamField>

### Comportamiento

- No elimina físicamente el registro
- Establece el campo `activo` en `false`
- La ruta no aparecerá en el listado de rutas activas
- Los clientes asignados mantienen su relación histórica

<CodeGroup>

```bash Solicitud
curl -X DELETE "https://api.fabricamarie.com/api/rutas/5" \
  -H "Authorization: Bearer {token}"
```

```json Respuesta 200 OK
{
  "message": "Ruta desactivada"
}
```

</CodeGroup>

## Asignar Clientes a Ruta

<Note>
  Asigna múltiples clientes a una ruta con orden específico para la secuencia de visitas.
</Note>

### Endpoint

```
POST /api/rutas/{rutaId}/asignar-clientes
```

### Autenticación

<ParamField header="Authorization" type="string" required>
  Token de autenticación Bearer
</ParamField>

### Parámetros de Ruta

<ParamField path="rutaId" type="integer" required>
  ID de la ruta a la que se asignarán los clientes
</ParamField>

### Parámetros del Body

<ParamField body="clientes" type="array" required>
  Lista de clientes a asignar con su orden
  
  <Expandable title="Estructura de cada elemento">
    <ParamField body="clientes[].cliente_id" type="integer" required>
      ID del cliente a asignar
    </ParamField>
    
    <ParamField body="clientes[].orden" type="integer" required>
      Orden de visita del cliente en la ruta (1, 2, 3...)
    </ParamField>
  </Expandable>
</ParamField>

### Comportamiento

- Utiliza `sync()` de Laravel, por lo que reemplaza completamente la asignación anterior
- Los clientes no incluidos en la solicitud serán removidos de la ruta
- El campo `orden` determina la secuencia de visitas del vendedor
- Se pueden asignar múltiples clientes en una sola operación

<CodeGroup>

```json Ejemplo de solicitud
{
  "clientes": [
    {
      "cliente_id": 45,
      "orden": 1
    },
    {
      "cliente_id": 46,
      "orden": 2
    },
    {
      "cliente_id": 47,
      "orden": 3
    },
    {
      "cliente_id": 50,
      "orden": 4
    }
  ]
}
```

```bash Solicitud completa
curl -X POST "https://api.fabricamarie.com/api/rutas/5/asignar-clientes" \
  -H "Authorization: Bearer {token}" \
  -H "Content-Type: application/json" \
  -d '{
    "clientes": [
      {"cliente_id": 45, "orden": 1},
      {"cliente_id": 46, "orden": 2},
      {"cliente_id": 47, "orden": 3}
    ]
  }'
```

```json Respuesta 200 OK
{
  "message": "Clientes asignados a la ruta"
}
```

```json Error 404 - Ruta no encontrada
{
  "message": "No query results for model [App\\Models\\Ruta] 5"
}
```

</CodeGroup>

## Casos de Uso

### Organizar ruta de visitas diarias

Asigna clientes en orden geográfico para optimizar el recorrido:

```json
{
  "clientes": [
    {"cliente_id": 45, "orden": 1},  // Primer cliente del día
    {"cliente_id": 46, "orden": 2},  // Segundo cliente
    {"cliente_id": 50, "orden": 3}   // Último cliente
  ]
}
```

### Reorganizar una ruta existente

Cambiar completamente la asignación de clientes:

```
POST /api/rutas/5/asignar-clientes
```

### Consultar clientes de una ruta ordenados

```
GET /api/rutas/5
```

Los clientes se devolverán con su campo `pivot.orden` para mostrar la secuencia.

## Relación con Tabla Pivot

La asignación de clientes a rutas se almacena en una tabla pivot con la estructura:

```
cliente_ruta
├── cliente_id
├── ruta_id
└── orden
```

El campo `orden` permite:
- Secuenciar las visitas del vendedor
- Optimizar rutas geográficas
- Priorizar clientes importantes
- Facilitar la planificación de entregas
